1️⃣ Firm Type Detection Layer ✅
Data Structure:
@dataclass
class FirmProfile:
    firm: str
    base_url: str
    detected_types: list[str]  # Multiple types allowed
    signals: list[str]          # Evidence used for classification
Detection Types Implemented:
- XML_SITEMAP - Valid XML sitemap available
- HTML_SITEMAP - HTML-based sitemap available
- DIRECTORY_HTML - Server-rendered directory with profile links
- DIRECTORY_FILTERED - Filters present (practice/office)
- SPA_API - XHR/JSON people data detected
- HARD_CASE - No recognizable patterns
Detection Logic:
1. Check robots.txt + sitemap URLs (XML vs HTML)
2. If no sitemap: Probe directory pages with Playwright
3. Detect API calls, DOM links, filters
4. Classify and persist to {firm}_profile.json
Example Outputs:
Latham & Watkins (XML_SITEMAP):
{
  firm: Latham & Watkins,
  base_url: https://www.lw.com,
  detected_types: [XML_SITEMAP],
  signals: [
    Checking 1 sitemap(s),
    Valid XML sitemap found
  ]
}
Kirkland & Ellis (HARD_CASE):
{
  firm: Kirkland & Ellis,
  base_url: https://www.kirkland.com,
  detected_types: [HARD_CASE],
  signals: [
    Checking 2 sitemap(s),
    No valid sitemap found,
    No profile links found in directory pages,
    No recognizable patterns detected
  ]
}

2️⃣ Strategy Router ✅
Data-Driven Strategy Selection:
IF XML_SITEMAP      → xml_sitemap
ELIF HTML_SITEMAP   → html_sitemap
ELIF SPA_API        → api_enumeration
ELIF DIRECTORY_FILTERED → filter_enumeration
ELIF DIRECTORY_HTML → dom_exhaustion
ELSE → hard_case_fallback
Automatic Routing:
- No hardcoded firm-specific logic
- Multiple strategies can run if coverage insufficient
- Strategies attempt in priority order (fast → slow)
---
3️⃣ URL Discovery Only ✅
Critical Separation:
- Phase 1-4: Collect profile URLs (no metadata fetching)
- Phase 5: Enrich URLs with metadata (separate, parallelized)
Benefits:
- Fast discovery (no HTTP overhead during collection)
- Retryable (can re-enrich same URLs later)
- Clear progress tracking (URLs vs attorneys)

4️⃣ Coverage Validation Engine ✅
Data Structure:
@dataclass
class CoverageReport:
    firm: str
    expected_total: int | None
    discovered_urls: int
    strategies_attempted: list[str]
    final_status: str  # SUCCESS | PARTIAL | HARD_CASE
    notes: list[str]
    url_sources: dict[str, int]  # strategy -> count
Validation Logic:
IF expected_total known:
    coverage = discovered / expected_total
    IF coverage >= 0.98 → SUCCESS
    ELIF coverage >= 0.5 → PARTIAL
    ELSE → HARD_CASE
ELSE:
    IF discovered >= 5 → SUCCESS
    ELSE → PARTIAL
Fallback Triggering:
- After each strategy, check coverage
- If insufficient, execute next strategy
- Stop when coverage threshold met or strategies exhausted
Example Output (Latham):
{
  firm: Latham & Watkins,
  expected_total: null,
  discovered_urls: 4206,
  coverage_ratio: null,
  strategies_attempted: [xml_sitemap],
  final_status: SUCCESS,
  notes: [Discovered 4206 attorneys (no expected total available)],
  url_sources: {xml_sitemap: 4206}
}

5️⃣ HTML Sitemap Support ✅
Implementation:
1. Try XML parsing first
2. If ET.ParseError → classify as HTML
3. Extract <a href> tags with regex
4. Filter for profile-like URLs
Code:
def _classify_sitemap(self, sitemap_url: str, domain: str) -> str | None:
    try:
        ET.fromstring(content)
        return "XML"
    except ET.ParseError:
        if b"<html" in content.lower():
            return "HTML"
