===================================================

Date: 2026-04-08

## What Changed

The output saving block in `run()` was moved from inside the `try` body into the
`finally` block. On the normal path (no exception), the flow is:

1. Firm loop completes
2. `_run_exception = None` is set
3. `except` block is NOT entered
4. `finally` block runs:
   - Excel wb.save() — writes the attorneys workbook
   - Source failure report — if any failures occurred
   - Firm summary CSV — via firm_summary_writer.write()
   - Coverage metrics JSON — via coverage_engine.save_run_metrics()
   - JSONL flush + close
   - Signal handler restored, refs cleaned
   - `_run_exception is None` → no re-raise
5. `return total_processed` executes normally

## Verification

- 61/61 tests pass (pytest tests/ -v)
- Output saving code is identical to before, just wrapped in individual
  try/except blocks within the finally to ensure each save operation
  is independent (one failure doesn't block others)
- The `return total_processed` statement is OUTSIDE the try/finally,
  so it only executes on the normal path (not after an exception re-raise)
