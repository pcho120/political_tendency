## Crash Safety Design

### try/finally in run()

Structure:
```
try:
    ...firm loop...
    _run_exception = None
except Exception as exc:
    _run_exception = exc
    logger.error(...)
finally:
    # Each save wrapped in its own try/except:
    1. Excel wb.save()
    2. Source failure report
    3. Firm summary CSV
    4. Coverage metrics JSON
    5. JSONL flush + close
    6. Signal handler restore + ref cleanup
    7. Re-raise _run_exception if set
```

Key properties:
- Each save operation is independent — if Excel save fails, JSONL still flushes
- The original exception is re-raised after all saves complete, so callers
  still see the error (not silently swallowed)
- JSONL is flushed per-attorney during the loop (line 1469), so even mid-firm
  crashes have most data already on disk; the finally just ensures final flush
- `wb` variable uses `# type: ignore[possibly-undefined]` since if the exception
  happens before wb is created, the save will fail gracefully (caught by the
  try/except wrapper)

### _rebuild_excel_from_jsonl() static method

Location: AttorneyFinder._rebuild_excel_from_jsonl(jsonl_path, excel_path)
Purpose: Post-crash recovery — rebuilds Excel from JSONL data

Column structure matches run():
  Firm, Attorney Name, Title, Offices, Departments, Practice Areas,
  Industries, Bar Admissions, Education, Extraction Status, Missing Fields,
  Profile URL, Data Source

Field mapping from JSONL record keys:
  firm → Firm
  full_name → Attorney Name
  title → Title
  offices → Offices (list→str)
  department → Departments (list→str)
  practice_areas → Practice Areas (list→str)
  industries → Industries (list→str)
  bar_admissions → Bar Admissions (list→str)
  education → Education (JSON-serialized)
  extraction_status → Extraction Status
  missing_fields → Missing Fields (list→str)
  profile_url → Profile URL
  data_source (or default 'firm_website') → Data Source

Error handling:
- Malformed JSONL lines are skipped with a warning
- If the JSONL file doesn't exist, returns 0
- If Excel save fails, logs error and continues
